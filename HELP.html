Cardnager

A code editing command center built for AI-assisted development. Treats your code like Lego bricks — every named block becomes a collapsible card you can copy, paste, navigate, and manage without ever scrolling through a raw file.

⬡ What is Cardnager?

The core idea behind the app and why it exists.

When you work with AI to write code, the AI gives you blocks — a function, a style section, a component. The problem is that your file is one long scroll. Finding the right place to paste, checking what's already there, and keeping track of what changed is slow and error-prone.

Cardnager solves this by parsing your file and turning every named block into a visual card. Each card shows the block's name, type, line count, and line range. You expand a card to see its code, and you paste directly into it — the app handles updating the file correctly.

The golden rule: Cardnager only works with files that use block markers. Before you use the app, your code needs to be structured with region, section, and segment comments. The AI Prompts feature helps you set this up automatically.

🕓 Recent Files

Quickly reopen files you have worked on before.

When Cardnager launches with no files open, or when you close all tabs, the Recent Files screen appears automatically. It lists the last 20 files you opened, sorted by most recently opened first. Each entry shows the file name, its full path, and when it was last opened.

Opening a Recent File

Click any file in the list to open it immediately. You can also double-click the entry. The file is added to a new tab and the recent files screen closes.

Opening Multiple Recent Files

Tick the checkboxes on the left of each entry to select multiple files, then click Open Selected to open them all at once, each in its own tab.

Removing Entries

Click the ✕ button on any entry to remove it from the list without opening it. Use Clear All to wipe the entire history. Files that no longer exist on disk are removed from the list automatically the next time the screen loads.

Last Opened Time

The timestamp shown on each entry reflects the last time you opened that file in Cardnager — either manually from the file picker, from the recent files list, or via session restore on startup. The time is displayed as a relative string such as 5 mins ago, 3 days ago, or 2 weeks ago.

Session restore counts as an open. When Cardnager starts and restores your previous tabs automatically, all restored files have their timestamp updated to the current time. This keeps the recent files list accurate to when you actually last used each file.

⌗ Block Marker Syntax

How to mark up your code so Cardnager can parse it into cards.

Every block needs an opening marker and a closing marker. The format depends on the language you're writing. Block names are the permanent identity of every block — use short, descriptive names. Never use numbers in names.

Always use names, never numbers. Cardnager shows numbered badges on cards (REGION 1, SECTION 2 etc.) but these are display-only and change automatically when blocks are added or removed. Always refer to a block by its exact name.

JavaScript / TypeScript

// #region Application Logic // #section Tab Management // #segment Open & Close function openTab() { ... } // #endsegment // #endsection // #endregion

CSS / SCSS

/* #region Application Styles */ /* #section Header & Navigation */ #header { ... } /* #segment Button Styles */ .btn { ... } /* #endsegment */ /* #endsection */ /* #endregion */

HTML

<!-- #region User Interface --> <!-- #section Header Toolbar --> <div id="header"> ... </div> <!-- #endsection --> <!-- #endregion -->

Other Languages

Language Opening Marker Closing Marker
Python # #region Name # #endregion
C# // #region Name // #endregion
SQL -- #region Name -- #endregion

The same pattern applies to #section / #endsection and #segment / #endsegment in every language.

◈ Card Hierarchy

The four card types and how they nest inside each other.

Region Section Segment Fragment
REGION —— Top-level group. A major area of your file. │ ├── SECTION —— Mid-level group. A class, a feature, a panel. │ │ │ ├── SEGMENT —— Small unit. One function or block of logic. │ │ │ └── FRAGMENT —— FRAGMENT —— Unmarked code (6+ lines) caught between named blocks. │ └── SECTION —— Another section in the same region.

Fragments

Fragments are created automatically. Any code that sits between named blocks but has no markers of its own becomes a Fragment card. Short fragments (5 lines or less) display as naked inline code with no card header. Longer fragments show as a collapsible Fragment card with the first line as its title.

Tip: Aim to wrap everything in named blocks. The fewer stray fragments you have, the cleaner and more navigable your file becomes.

📁 Opening Files

Three ways to open files in Cardnager.

Recent Files Screen

Every time you launch the app with no files open, the Recent Files screen appears. It lists the last 20 files you worked on, sorted by most recent. You can click any file to open it, double-click to open it immediately, or select multiple files with checkboxes and open them all at once.

Browse Files

Click the + button in the tab bar, or go to File → Open File, or press Ctrl+O. The native file dialog opens and supports multi-file selection — hold Ctrl to select several files at once.

Drag & Drop

On the Recent Files screen, drag any file from your file explorer and drop it onto the drop zone at the bottom. Multiple files can be dropped at once.

Session restore: When you close Cardnager with files open, it remembers which files were open and restores them automatically on the next launch.

◫ Tab Management

Work with multiple files simultaneously using the tab bar.

Each open file gets its own tab at the top of the window. Click a tab to switch to it. The app remembers scroll position and expanded cards separately for each tab.

  • Unsaved indicator: A tab with unsaved changes shows an asterisk (*) in its title and the Save button turns orange.
  • Closing a tab: Click the × on the tab. If the file has unsaved changes, a confirmation dialog appears before closing.
  • New tab: Press Ctrl+N or use File → New Tab to create a blank untitled file.
On close: If you try to quit the app with unsaved files, Cardnager intercepts the close and asks you to confirm. Your work is never silently lost.

▣ Working With Cards

How to expand, collapse, edit, and interact with cards.

Expanding & Collapsing

Click anywhere on a card header to expand or collapse it. When a card has child cards inside (for example, a Region containing Sections), expanding it reveals the children. Expanding a leaf card (one with no children) reveals the code editor directly.

Editing Code

Expand any leaf card to reveal a code editor. Click inside and start typing. The editor auto-resizes as you type. Changes are tracked immediately — the Save button activates as soon as you make any edit.

Line Numbers

Every editor shows real file line numbers on the left, not relative numbers. These update live as you edit and as other blocks above shift.

The Card Menu ⋮

Every card header has a three-dot menu button on the right. Click it to access:

  • Pin / Unpin: Add or remove the card from the Pinned Sidebar.
  • Add Region / Section / Segment Above: Insert a new empty block directly above this card. Only valid block types for the current position are enabled.
  • Add Region / Section / Segment Below: Insert a new empty block directly below this card.
  • Delete Block: Remove this block and all its contents from the file. A confirmation dialog appears first.
Strict hierarchy rules: The menu only enables Add options that make structural sense. You cannot add a Region inside a Segment, for example. Invalid options appear greyed out.

Refresh ↻

The ↻ button in the toolbar re-renders the entire card tree from the current in-memory file content. Use it if the display ever looks out of sync after a heavy paste, undo, or structural operation. It does not reload the file from disk — it re-parses what is already loaded. The button plays a brief spin animation to confirm it fired.

Sticky Headers

When you scroll through a large expanded card, its header sticks to the top of the viewport so you always know which block you are reading. The deepest active card wins the sticky position.

⚠ Duplicate Name Warning

Automatic detection of blocks that share the same name.

Every block name in a file must be unique. Duplicate names confuse the parser, break Code Ops matching, and cause unpredictable behaviour when pasting. Cardnager watches for this automatically.

When It Triggers

The duplicate warning bar appears at the top of the card list in two situations:

  • On file open — every time a file is loaded or a tab is switched to, the entire block tree is scanned for name collisions.
  • On refresh — every time you click the ↻ Refresh button, the scan runs again on the current content.

What It Shows

The orange bar displays a summary line such as "⚠ 4 blocks share 2 duplicate names" followed by a row of pills, one per duplicate name. Each pill shows the name and how many times it appears — for example "App Init" ×2. Hovering a pill shows a tooltip with the block type (region, section, or segment).

Fix duplicates before doing any Code Ops work. If two blocks share a name, a scan may match the wrong one and apply changes to the incorrect location.

Dismissing the Warning

Click the ✕ button on the right of the bar to dismiss it for the current session. It will reappear the next time the file is opened or refreshed if duplicates still exist.

Fixing a Duplicate

Expand the card whose name you want to change. Edit the marker comment directly in the code editor — for example change // #region App Init to // #region App Ready — then click ↻ Refresh. If the collision is resolved, the warning bar disappears automatically.

Paste-Time Check

In addition to the bar, a separate duplicate check runs every time you use the PASTE button on a card. If the incoming block has the same name as an existing block elsewhere in the file, a native confirmation dialog appears before anything is written, giving you the option to cancel.

⧉ Copy & Paste

The core workflow — getting AI code into the right block.

Copy Button

Every card has a COPY button. Clicking it copies the entire block — including its markers and all child content — to your clipboard. The button flashes green to confirm.

Paste Button

Every card also has a PASTE button. This is the main way to update a block with new AI-generated code. When you click Paste, Cardnager runs four safety checks before writing anything:

  1. Identity check. If the clipboard content is identical to what's already in the block, the paste is silently skipped. No pointless overwrites.
  2. Mismatch check. If you paste a Section into a Region, or raw code into a structured block, a warning dialog explains the issue and asks you to confirm.
  3. Duplicate check. If a block with the same name already exists elsewhere in the file, you are warned. Duplicate names break the parser.
  4. Hierarchy check. If the pasted block type would violate the nesting rules (e.g. pasting a Region inside a Segment), a warning appears before anything is written.

If all checks pass, the block is replaced and the file is updated. The card flashes green and the view scrolls to keep the pasted card in position.

Structural vs surgical paste: If the content you are pasting contains no block markers, Cardnager uses a fast surgical path — it updates just that block's lines without rebuilding the entire card tree, which is much faster for large files.

💾 Saving & Backups

How saving works and where backups go.

Save

Press Ctrl+S or click the orange SAVE* button. Before writing, Cardnager cleans up triple blank lines automatically. After a successful save, the button flashes green and the tab title loses its asterisk.

Save As

Use File → Save As or press Ctrl+Shift+S to save the current file to a new path and name.

Save All

Use File → Save All to save every dirty tab at once.

Automatic Backups

Every time you save, Cardnager automatically creates a timestamped backup of the previous version. Backups are stored in a _backups folder inside the same directory as your file.

Backup filename format: filename_BACKUP_2026-01-30_14-30-05.ext

Restoring a Backup

Go to Settings → Restore Backup. A modal lists all backups for the currently active file, sorted newest first with timestamps and file sizes. Click RESTORE on any backup to load it into the editor. A confirmation dialog warns you that this overwrites your current content. The restore is also added to the undo stack, so you can press Ctrl+Z to undo a mistaken restore.

Restore Backup is greyed out in the menu if the current file has no _backups folder yet.

↩ Undo & Redo

Structural undo that tracks block-level operations, not single keystrokes.

The ↩ and ↪ buttons in the toolbar control Cardnager's structural history. This is separate from native textarea undo — it snapshots the entire file content before major operations like paste, delete, add block, and replace.

  • Undo: Ctrl+Z or click ↩ — rolls back to the last snapshot.
  • Redo: Ctrl+Y or click ↪ — re-applies the rolled-back change.
The undo stack holds up to 50 snapshots per tab. Native textarea undo (Ctrl+Z while typing inside a code editor) still works independently for fine-grained edits within a single block.

⌕ Search System

Find blocks by title or search through code content.

Search by Title

Select By Title in the dropdown next to the search box. As you type, all cards whose names do not match the query are hidden. Parent cards that contain a match are automatically expanded and kept visible. This is the fastest way to jump to a specific block by name.

Search by Code

Select By Code to search the actual content of every code block. All matches across all blocks are collected and counted. Use the ▲ and ▼ navigation buttons to jump between matches one by one — the app expands the containing cards and scrolls to each match automatically.

Replace

When a code search returns results, a Replace button appears next to the nav buttons. Click it to open the replace panel, which offers two options:

  • Paste Clipboard: Replaces all matches with whatever is currently on your clipboard.
  • Type text: Type a replacement string manually and click GO.
Replace affects all matches across all open cards at once. Use it carefully. It is undoable via Ctrl+Z.

⊞ Fold Controls

Collapse or expand every card in the file at once.

The ▼ and ▲ buttons in the toolbar control all cards globally.

  • ▼ Expand All: Opens every card at every level so all code is visible.
  • ▲ Collapse All: Closes every card so you see only the top-level summary view.

The same actions are available under View → Expand All Blocks and View → Collapse All Blocks.

⊟ Overview

A bird's-eye view of your entire file structure.

Click the OVERVIEW button in the toolbar to open an overlay showing every Region, Section, and Segment as a small card arranged in columns — one column per Region. Each card shows its type, name, and line count.

Navigation

Click any card in the overview to close the overview and jump directly to that block in the main view. The target block is highlighted with a blue glow on arrival.

Zoom

Scroll the mouse wheel while hovering over the overview to zoom in and out. Zoom is centered on the mouse position. The current zoom level is shown in the top bar.

Pan

Hold the middle mouse button and drag to pan the overview canvas freely.

Tooltips

Hover over any overview card to see a tooltip with the block's full name, type label, line count, and line range.

Press Esc or click the ✕ button to close the overview.

📌 Pinned Sidebar

Keep your most-used blocks one click away at all times.

Any card can be pinned to a persistent sidebar on the right side of the app. Pinned cards stay visible while you scroll through the rest of your file — useful for keeping a reference block, a constants section, or a frequently edited function always accessible.

Pinning a Card

Open the card's ⋮ menu and click 📌 Pin. The sidebar appears and the card is added to it.

Jumping to a Pinned Card

Click any card in the sidebar to instantly scroll the main view to that card. If the card is inside a collapsed parent, all parent cards are expanded automatically before scrolling.

Unpinning

Click the 🗑 button on any pinned card, or open the card's ⋮ menu and click 📌 Unpin. When the last pinned card is removed, the sidebar closes automatically.

Resizing the Sidebar

Drag the thin vertical handle between the main content and the sidebar to resize it. The width is saved and restored between sessions.

Per-tab pins: Pinned cards are stored per file. Switching tabs shows the pins for that tab. Pins are saved to your session config and restored on next launch.

≡ Raw View

Switch between the card view and a plain text editor.

Go to View → Toggle Raw View to switch the current tab from card view into a full plain-text editor showing the raw file content. A checkmark in the menu indicates raw mode is active. The fold buttons are hidden in raw mode since they have no function there.

While in raw view, all edits are tracked live and the Save button activates normally. Switching back to card view commits the raw content and re-parses it into cards immediately.

When to use raw view: Useful when you need to do a large structural edit — like adding several new regions at once — that would be awkward to do card by card. Edit the raw markers, switch back, and the new cards appear.

🛡 Safety & File Integrity

Every layer of protection Cardnager puts between you and a broken file.

The entire point of Cardnager is to let you apply AI-generated code changes quickly and confidently. Every feature that writes to your file is wrapped in at least one safety layer. This section lists all of them in one place.

Nothing writes to disk without your action. All edits happen in memory first. The file on disk only changes when you explicitly save. Every write operation is also backed by undo and a backup.

1 — Paste Safety Checks

Every time you click the PASTE button on a card, Cardnager runs four checks in sequence before writing a single character:

  • Identity check. If the clipboard content is byte-for-byte identical to what is already in the block, the paste is silently skipped. No pointless overwrites, no dirty flag, no undo entry.
  • Mismatch check. If you paste a Section into a Region slot, or raw code into a structured block, a warning dialog explains the conflict and lets you cancel before anything changes.
  • Duplicate name check. If the incoming block shares a name with a block that already exists elsewhere in the file, a warning dialog appears. You can cancel or proceed knowingly.
  • Hierarchy check. If the pasted block type would violate nesting rules — for example pasting a Region inside a Segment — a warning stops the operation before it starts.

All four checks must pass before the block is written. If any check fails and you choose to cancel, the file is left completely unchanged.

2 — Duplicate Name Warning Bar

On every file open and every refresh, the entire block tree is scanned for duplicate names. If any are found, an orange warning bar appears at the top of the card list showing every duplicate name and how many times it appears. This catches naming collisions introduced by manual editing or by pasting raw content directly.

See the Duplicate Name Warning section for full details.

3 — Automatic Backups on Every Save

Every time you save a file, Cardnager automatically copies the previous version to a _backups folder inside the same directory before writing the new content. The backup filename includes a full timestamp so you can always identify exactly when it was made.

Backups can be browsed and restored at any time via Settings → Restore Backup. A confirmation dialog appears before any restore, and the restore operation itself is pushed onto the undo stack so it can be rolled back with Ctrl+Z.

4 — Structural Undo Stack

Cardnager maintains a dedicated undo history separate from native textarea undo. A full snapshot of the file content is pushed onto the stack before every major write operation:

  • Every PASTE button action
  • Every Code Ops operation — Replace, Insert Before, Insert After, Remove, and all "All" variants
  • Every Add Block action from the ⋮ card menu
  • Every Delete Block action
  • Every Restore Backup action

The stack holds up to 50 snapshots per tab. Press Ctrl+Z to roll back or Ctrl+Y to redo. Even if you apply ten Code Ops operations in a row, you can step back through all of them one by one.

5 — Confirmation Dialogs

Any action that is destructive or hard to reverse triggers a native confirmation dialog before executing. You must click Yes explicitly — these cannot be dismissed by accident:

  • Delete Block — Removes a card and all its content from the file.
  • Paste All Content — Replaces the entire file with whatever is on the clipboard.
  • Restore Backup — Overwrites the current editor content with a backup version.
  • Close Tab with Unsaved Changes — Warns you before discarding changes on a dirty tab.
  • Paste mismatch / duplicate / hierarchy violations — Each of the four paste safety checks uses its own dialog so you can cancel.

6 — Unsaved Changes Protection

Cardnager intercepts the window close event before the app quits. If any open tab has unsaved changes, a dialog tells you how many dirty files are open and warns that closing will lose them permanently. You can cancel the close, save first, and then exit cleanly. The app will never quit silently while you have unsaved work.

The Save button is your signal. When any tab has unsaved changes its title gains an asterisk and the Save button turns orange. A clean file shows no asterisk and a greyed-out Save button. Always check before closing.

⚙ Code Ops

Find, replace, insert, and remove code across your entire file with surgical precision.

Click the CODE OPS button in the toolbar to open the Code Ops panel. It operates in four modes.

⇄ Replace
Replace Mode
Find exact code and replace it with new code. The matched lines are removed and the replacement is written in their place, preserving surrounding indentation.
▲ Insert Before
Insert Before Mode
Find a block of code and insert new code immediately before it without removing the original. Useful for adding imports or inserting a function before another.
▼ Insert After
Insert After Mode
Find a block of code and insert new code immediately after it. Useful for adding a new function after an existing one or appending lines to a block.
✕ Remove
Remove Mode
Find a line or block of code and delete it entirely from the file. No replacement needed — the matched lines are simply removed and the file is stitched back together.

How to Use (Replace, Insert Before, Insert After)

  1. Select a mode from the mode bar at the top.
  2. Paste the code you want to find into the Find This Code field.
  3. Paste or type the replacement or insertion code into the Replace With field.
  4. Click SCAN. The app searches the entire file and reports how many matches were found.
  5. Use the ▲ ▼ buttons to step through each match. The preview panel shows the surrounding lines with the match highlighted in orange.
  6. Click Replace This to apply the operation to the current match only, or Replace All to apply to every match at once. After applying, the preview updates to show the result highlighted in green.

How to Use (Remove)

  1. Select the ✕ Remove mode. The second input field is hidden — you only need the Find field.
  2. Paste the line or block of code you want to delete into the Find This Code field.
  3. Click SCAN. Matches are found and highlighted in the preview.
  4. Click Remove This to delete the current match, or Remove All to delete every match at once.

Before / After Preview

Every time you apply an operation, the preview panel updates to show a diff-style result so you can verify the change before moving on:

  • Replace & Insert modes — The newly written lines are shown highlighted in green with their new line numbers. Surrounding context lines are shown in normal colour.
  • Remove mode — The deleted lines are shown in red with a strikethrough and a − marker in the gutter. The lines above and below the deletion point are shown in normal colour so you can confirm the file joined correctly.

Clearing the Inputs

Each input field has a small ✕ button in the top-right corner. Click it to clear that field instantly so you can paste new code without having to select-all and delete manually.

The Clear All button in the top-right of the Find field header clears both fields at once and fully resets the scan state — matches, preview, and action buttons — so you are back to a clean slate in one click.

Smart Matching

Code Ops uses token-based matching, which means it ignores formatting differences like extra spaces or different indentation. It collapses all whitespace before comparing, so a match is found even if your clipboard content has slightly different spacing than what's in the file.

Indentation Preservation

When replacing or inserting, Code Ops automatically normalizes the indentation of your replacement code to match the indentation level of the target location in the file. You do not need to manually adjust tabs or spaces.

Code Ops is undoable. A snapshot is pushed to the undo stack before every operation.

✦ AI Prompts

Ready-made prompts engineered specifically for working with block-structured code.

Click the AI PROMPTS button in the toolbar to open the prompts panel. It contains a library of prompts you can copy and paste directly into any AI assistant.

Project Types

Select your project type from the dropdown at the top of the panel. The prompts automatically include the correct block marker syntax for that project type, so the AI will produce code that Cardnager can parse correctly.

Electron App
JS + HTML + CSS markers. Ideal for Electron desktop apps like Cardnager itself.
Web App
JS + HTML + CSS markers. For browser-based projects.
Python
Python comment markers. For scripts and backend apps.
Node.js
JS markers only. For server-side JavaScript projects.
C# Desktop
C# markers. For .NET desktop applications.

Available Prompts

  • Getting Started: How to Use Cardnager with AI — Sets up the AI with the core collaboration rules: complete code only, touch nothing unmentioned, address blocks by name never by number, and the correct code delivery format. Also injects project-specific context such as Electron IPC details for Electron projects. Use this at the start of every new AI conversation before sending any code.
  • Structure My Code into Blocks — Ask the AI to reorganize existing unstructured code into Cardnager block format. The AI will add the correct markers for your project type, apply proper naming rules, and preserve all existing logic without changing a single line of it.
  • Break Down a Block into Sub-Levels — Use when a block is too large and needs to be split into nested sub-blocks. The AI inserts a new level of hierarchy inside the existing block without renaming the outer block or changing any code.
  • Split a Block into Multiple Blocks — Use when a block should become two or more independent blocks of the same type. The AI divides the content, names each new block descriptively, places them all at the same nesting level, and preserves all existing logic.

Click the arrow on any prompt card to expand and read the full prompt text. Click COPY to copy it to your clipboard, then paste it into your AI assistant followed by the code you want to work with.

How the AI Should Deliver Code

The Getting Started prompt instructs the AI to follow a specific delivery format depending on the size of the change. Understanding this helps you know what to expect:

  • Small changes — a few lines inside a large segment or section — the AI delivers a Find / Replace With pair. Copy the Find text into Code Ops' Find field, copy the Replace With text into the Replace With field, then click Scan and Replace.
  • Large changes — most or all of a segment or section needs to be rewritten — the AI delivers the full named block directly. Copy the entire block and paste it into the correct card using the PASTE button. No Code Ops needed.
Never use position numbers. The prompt enforces that the AI always refers to blocks by their exact name (e.g. "find the section named Login Logic"), never by a number (e.g. "find section 3"). Numbers change every time a block is added or removed. Names are permanent.

>_ Terminal Launcher

Open a terminal directly in the folder of the active file.

The terminal launcher sits in the top-right of the toolbar. Select a terminal type from the dropdown, then click the >_ button to launch it. The terminal opens at the directory of the currently active file — no manual navigation needed.

Supported Terminals

  • PowerShell — Windows PowerShell
  • CMD — Windows Command Prompt
  • Git Bash — Requires Git for Windows to be installed

Your terminal preference is saved automatically and restored on next launch. The terminal toolbar can be hidden via View → Show Terminal.

◐ Theme & Colors

Customize the visual appearance of the app.

Go to Settings → Theme Colors to open the theme editor.

Base Themes

Select from three base themes using the Overall Theme dropdown. Changing the theme updates all color pickers to that theme's defaults, which you can then further customize.

  • Grey (Default) — Medium grey backgrounds. The default Cardnager look.
  • Dark — Near-black backgrounds for very dark environments.
  • Light — White and light grey backgrounds with softened card colors.

Custom Colors

Every card type has its own color customization: badge and border color, and badge text color. You can also customize the UI text, code text, and card title text colors independently.

Saving & Resetting

Click Save Changes to apply and persist the theme. Click Reset to Defaults to restore the Grey theme. Changes take effect immediately without restarting the app.

T Typography

Control the fonts used throughout the app.

Go to Settings → Typography to open the font editor. Four font slots can be set independently:

  • UI Font — Menus, buttons, labels, and all interface text.
  • Code Editor Font — The font used inside all code block editors.
  • Card Titles — The font used for block names displayed on card headers.
  • Card Type Labels — The font used for REGION / SECTION / SEGMENT badges.

Zoom

The code editor font size can be changed using zoom controls without opening the typography modal:

  • Ctrl+Plus — Increase font size
  • Ctrl+Minus — Decrease font size
  • Ctrl+0 — Reset to default size (12px)

Zoom is also available under View → Zoom In / Zoom Out / Reset Zoom.

▬ Status Bar

Live stats for the currently active file.

The status bar at the bottom of the window shows five counters that update every time the file is parsed:

Regions
Total number of top-level Region blocks in the file.
Sections
Total number of Section blocks across all Regions.
Segments
Total number of Segment blocks across all Sections.
Fragments
Number of unmarked code fragments auto-detected by the parser.

The total line count of the file is shown on the right side of the status bar.

The status bar can be hidden via View → Show Status Bar. Its visibility preference is saved between sessions.
